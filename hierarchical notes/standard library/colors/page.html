      • Color Component Storage Format:
        • 8, 16, 32-bit unsigned integer.
        • 8, 16, 32-bit signed integer.
        • 16, 32, 64-bit floating point. (normalized to [0,1])
    • Manipulation
      • Common interface in all cases. (e.g. you can get saturation from an object using RGB internal storage and set green for an object using CMY internal storage)
      • Support construction and deconstruction.
      • Allow assignment between color objects of different internal formats by automatic conversion.
      • Allow assignment between color components of different storage formats by automatic conversion.
      • Allow the .bgra OpenCL syntax.
    • Views
      • vector
      • array of integer or floating-point values (depending on the internal format). (for use with OpenGL API calls)
      • Arrays of colors should also be viewable as arrays of quadruples of integer or floating-point values respectively. (again for use with OpenGL API calls)
    • Operations
      • negative
    • Safety
      • Values are considered valid when they are within the representable range of the internal storage format for integer components (trying to assign 1024 to a byte component is still an error) and in the interval [0,1] for floating-point components.
      • Checks that can be done in compile-time (e.g. using compile-time-evaluation and/or value range propagation) are always on.
      • Checks that can't be done in compile-time are only on in debug mode, unless overriden by a compilation flag.
  • Standard Constants
    • Color names organized per naming standard (e.g. HTML color names, X11 color names, etc.)
    • Implementation can generate definitions automatically from corresponding standard documents.
