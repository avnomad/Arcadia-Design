Notes about the standard library in general:
  • Ideas
    • Provide a way to clear a buffer that guarantees the clear will take place and won't be optimized out. e.g. passwords before returning from a function. D support that? Can be attribute or method.
    • Include a library with probability and statistics related functions? If yes, P(A|B) syntax is available for use.
    • Include a Cache (or Memoize?) class that enables something to be evaluated once and used many times until it changes. Typically this class will be used to store the result of calling a function, using its arguments as the address and should support recursive calls.
    • Include an "asm" module that defines a C-like "asm" statement and other facilities for writing low-level architecture-specific code.
    • Include a module for facilitating creation of command-line applications similar to Python argparse library.
  • Naming Conventions
    • Ideas
      • Formal Parameters
        • A 'c' prefix could mean "count" and such a parameter would name the number of some objects.
        • An 's' prefix could mean "size" and such a parameter would name the size (in bytes) of an object.
      • Functions
        • algorithm[ed][i][s] (e.g. mappedi), where:
          • Optional "ed" suffix only applies to methods and is used for methods that do not modify the object on which they are invoked.
          • Optional "i" suffix is used for algorithms that also pass the element index to their function argument.
          • Optional "s" suffix is used for algorithms that also pass the container size to their function argument.
      • Iterators
        • Concept prefixes
          • i ⟶ input
          • o ⟶ output
          • f ⟶ forward
          • b ⟶ bidirectional
          • r ⟶ random access
