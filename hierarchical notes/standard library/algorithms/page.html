  • Modify algorithms to work with ranges instead of iterators.
  • Include 'smart' version of map (and other algorithms?) that will work recursively on nested containers based on the function argument type.
  • Algorithms should check their arguments for supported operations and adapt accordingly. e.g.
    • A find function could use binary search on sorted random-access ranges, but fall-back to linear search on forward ranges.
  • Most algorithms can be recast to transform ranges to other ranges without actually doing any work, then a copy algorithm, or an (deep) assignment can trigger the actual work.
