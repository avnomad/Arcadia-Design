    • Encode and decode strings to and from:
      • URL
      • MIME
      • Morse (doesn't need to be in the standard library, just needs to be implementable under the same interface)
      • other?
  • Between Strings and Other Types
    • Uniform mechanism for all cases.
    • From and to arithmetic types.
    • From and to type names.
      • Given a type you should be able to get its name as a string and given a string containing a type's name you should be able to get the object representing the type. (the type object will have methods to create values of this type if needed)
