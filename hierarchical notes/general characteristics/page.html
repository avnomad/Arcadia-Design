    • Core Language
      • File Format (e.g. text or binary source files?)
    • Module Language
      • Module File Format
    • Hardware Abstraction
    • Substitution Rules Language for Editors that can auto-complete/replace text as you type.
    • Presentation Rules Language for Editors that can present source code in a typeset format.

  • source files will be assumed to contain Unicode characters by default.
  • identifiers, strings, comments, etc. can contain any number and kind of Unicode characters.
  • variables can be stored in heap, static, stack. no Java-style references, a number of pointer types.
  • same function should be callable in different ways:
    • classic call
    • inline call
    • compile time call
    • virtual function call? (should it be different from "classic"?)
  • duality between (algebraic) structures containing interface and axioms and types that can conform or not to the structure.
  • type composition and decomposition
  • array generation operators:
    • 1...n
    • 1,3,...,7
    • [a,b]:0,5
    • 1,...,2ⁱ,...,2ⁿ
  • Classification
    • case-sensitive, free format, no preprocessor
    • procedural with functional influence.
    • statically typed (with type inference?) with dynamic elements (at least through dynamic dispatch).
  • General Direction
    • Clearly defined "under the hood" behaviour.
    • Clear and intuitive abstraction without 'holes'.
    • Vast meta-programming facilities.
    • Easily modifiable programs.
  • A rejected idea about the language name was "Experiment".
